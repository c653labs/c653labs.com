About

C653 Labs is a small software and DevOps consulting firm based in New York. We love working with our clients to build performant web-scale projects using the right tool for each job. Some our recent projects have worked with Docker, AWS, Python/Flask, and JavaScript/Node.js/Vue.js.

C653 Labs was started in 2017 and is owned and operated by Brett Langdon.

Featured work

Resume parsing & layout analysis

Problem

Client needed a way to both parse data from resumes as well as auto add/fix links embedded in the resumes.

Solution

We developed a resume parsing library to appropriately parse the text out of a resume. This parsing library used custom layout analysis processing that improved the quality of the text groupings parsed compared to other text extraction methods.

As part of this project we also developed a library to allow parsing the adding or fixing all links found embedded inside of the resumes.

Terraform AWS configuration

Problem

Client wanted an auditable & testable way to provision their AWS environment.

Solution

We configured Terraform to provision multiple AWS environments utilizing Continuous Integration to ensure consistent configuration standards, and Continuous Delivery to deploy any changes to their AWS environments.

Oauth 2 single sign-on service

Problem

Client was building a product that needed user authentication.

Solution

We developed a central shared single sign-on service for the client that allowed their users to re-use their login credentials between multiple applications.

E-mail open/click tracking

Problem

Client desired a way to more tightly integrate e-mail open and click tracking with their existing event tracking system.

Solution

We developed a custom event tracking service that supported both open tracking pixels and redirecting click tracking urls.

Application migration to Docker

Problem

Client wanted to move their applications to Docker.

Solution

We helped update their applications to allow running them in Docker containers, which included configuring 12-factor app environment variable configuration pattern.

We migrated their Jenkins testing environment to utilize Docker for running unit and integration tests.

We migrated their production environment to AWS ECS.
